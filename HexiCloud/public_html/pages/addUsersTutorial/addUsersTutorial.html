  1. Go to ‘My Services’
  2. Click ‘Users’
  3. Click the ‘Users’ tab
  4. Click ‘Add’
  5. Add user details
    • First name:
    • Last name:
    • Email address:
    • Username:
    • Manager’s email address:
  6. Choose your role
    • Identity Domain Administrator
    • Services
    • Entitlement Administrator
    • Bucket Entitlement Administrator
    • Service Instance Administrator
  7. Click ‘Add’
    • Once the user account has been created, Oracle Cloud sends a notification email containing the following sign-in credentials:
      • Username
      • A temporary password. Users must change this password the first time they sign in.
      • The identity domain name.
    • Users now have all the necessary sign-in credentials, but need the appropriate URLs. As such, the identity domain administrator or the service administrator must:
      • Provide users with the ‘My Services’ application URL
      • Provide users with one or more service URLs, depending on the services they’re allowed to access.
    • Identity domain administrators and service administrators can modify existing user accounts and change the roles assigned to each user. Roles control access to applications, resources, and services. If users forget their password, identity domain administrators can reset it.
